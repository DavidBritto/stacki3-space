Qué es

No es una colección de dotfiles. Es un recorrido diario.

La idea es reducir cambios de contexto: abrir, buscar, cambiar tema, revisar estado y volver al trabajo desde una superficie coherente.

Terminal primero

Kitty, Zsh y tmux sostienen el trabajo; i3 ordena sin robar protagonismo.

Menús con salida

`space`, Rofi y overlays TUI exponen acciones reales, no callejones sin salida.

Ruido bajo

Deep-space, Polybar compacta y Dunst oscuro mantienen el sistema visible pero quieto.

Instalación

APT instala el producto. Tú decides cuándo aplicar tu HOME.

El paquete coloca STACKI3-Space en `/usr/share/stacki3-space`. Tus dotfiles se actualizan solo cuando ejecutas `stacki3-space apply`. 

sudo install -d -m 0755 /etc/apt/keyrings
curl -fsSL https://davidbritto.github.io/stacki3-space/stacki3-space-archive-keyring.asc \
  | sudo tee /etc/apt/keyrings/stacki3-space.asc >/dev/null
echo "deb [arch=amd64 signed-by=/etc/apt/keyrings/stacki3-space.asc] https://davidbritto.github.io/stacki3-space stable main" \
  | sudo tee /etc/apt/sources.list.d/stacki3-space.list >/dev/null
sudo apt update
sudo apt install stacki3-space
stacki3-space apply --deps

Actualizaciones

Ideal para dogfooding: mejoras aquí, update allá.

Publicas una release, GitHub Pages sirve el repo APT, y tu notebook recibe la nueva versión con el flujo normal de paquetes.

  1. 1

    Mejoras el repo y publicas tag/release.

  2. 2

    GitHub Actions construye el `.deb` y actualiza `dists/` + `pool/`.

  3. 3

    El usuario corre `sudo apt update && sudo apt upgrade`.

  4. 4

    `stacki3-space apply` sincroniza el payload con backups.
